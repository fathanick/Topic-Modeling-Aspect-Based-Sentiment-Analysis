================================================================================
1. OVERVIEW
================================================================================

This module performs Aspect-Based Sentiment Analysis (ABSA) on Indonesian
tourism reviews using Large Language Model inference via Groq API. The approach
uses few-shot prompting with Llama 3.1 to identify tourism-specific aspects
and classify sentiment for each aspect mentioned in reviews.

Key Features:
- FREE API tier: 30 requests/min, 14,400 requests/day
- Ultra-fast inference via Groq's LPU (Language Processing Unit)
- Few-shot prompting for domain adaptation without fine-tuning
- Multi-aspect detection per review
- Evidence extraction for sentiment justification
- Batch processing for API efficiency

4.3 Batch Prompt Construction
-----------------------------
Reviews are grouped into batches for efficient processing:

def create_batch_prompt(reviews_batch):
    prompt = FEW_SHOT_USER  # Few-shot examples
    for i, review in enumerate(reviews_batch, 1):
        text = review.get('original_text', review.get('cleaned_text', ''))[:800]
        stars = review.get('stars', 'N/A')
        prompt += f'\nReview {i} ({stars} stars): "{text}"'
    prompt += "\n\nOutput:"
    return prompt

- Maximum text length per review: 800 characters (truncated if longer)
- Star rating included to provide context
- Reviews numbered for alignment with JSON output

   f. Wait for rate limit delay (3 seconds)

5.3 API Call Implementation
---------------------------
def analyze_batch(client, reviews_batch, results_tracker, retry_count=0):
    response = client.chat.completions.create(
        model=MODEL_NAME,
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": prompt}
        ],
        max_tokens=MAX_TOKENS,
        temperature=0.1  # Low temperature for consistent outputs
    )

Temperature Setting:
- Value: 0.1 (low)
- Rationale: Ensures consistent, deterministic sentiment classifications
- Higher values would introduce more variation in outputs

5.4 JSON Extraction
-------------------
The model response is parsed using multiple fallback strategies:

def extract_json_from_response(response_text):
    # Strategy 1: Remove markdown code blocks
    text = re.sub(r'```(?:json)?\s*', '', text)
    text = re.sub(r'```', '', text)

    # Strategy 2: Find JSON array pattern
    match = re.search(r'\[[\s\S]*\]', text)
    if match:
        json_str = match.group(0)
        # Clean trailing commas
        json_str = re.sub(r',\s*}', '}', json_str)
        json_str = re.sub(r',\s*]', ']', json_str)
        return json.loads(json_str)

    # Strategy 3: Find individual JSON objects
    for match in re.finditer(r'\{[^{}]*(?:"aspects"\s*:\s*\[[^\]]*\][^{}]*)?\}', text):
        obj = json.loads(match.group(0))
        if 'aspects' in obj:
            objects.append(obj)

5.5 Error Handling
------------------
Robust error handling with retry logic:

- Rate Limit Errors (429):
  * Wait 30 * (retry_count + 1) seconds
  * Exponential backoff up to 3 retries

- Other Errors:
  * Wait 5 * (retry_count + 1) seconds
  * Log error and continue
  * Mark review as 'failed' if all retries exhausted

================================================================================
7. SENTIMENT SCORE CALCULATION
================================================================================

Formula:
--------
sentiment_score = (positive_count - negative_count) / total_mentions

Interpretation:
---------------
+1.0  = All mentions are positive
 0.0  = Equal positive and negative, or all neutral
-1.0  = All mentions are negative

Score Ranges:
-------------
+0.7 to +1.0  : Excellent (strong positive sentiment)
+0.3 to +0.7  : Good (moderate positive sentiment)
 0.0 to +0.3  : Mixed (slight positive or balanced)
-0.3 to  0.0  : Concerning (slight negative)
-1.0 to -0.3  : Poor (strong negative sentiment)

11. LIMITATIONS AND CONSIDERATIONS
================================================================================

API Limitations:
----------------
- Free tier: 30 requests/minute, 14,400 requests/day
- Token limits per request
- Internet connection required

Model Limitations:
------------------
- May miss sarcasm or highly implicit sentiment
- Context window limits (text truncated to 800 chars)
- Occasional JSON formatting errors in response

Data Considerations:
--------------------
- Reviews in languages other than Indonesian may be misclassified
- Very short reviews may lack sufficient context
- Emojis are not explicitly handled

Mitigation Strategies:
----------------------
- Retry logic for API errors
- Multiple JSON parsing strategies
- Rate limiting to avoid API throttling
- Batch processing to reduce API calls
